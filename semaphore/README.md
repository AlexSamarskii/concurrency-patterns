**Семафор** — это механизм, ограничивающий количество горутин, которые могут одновременно получить доступ к общему ресурсу или выполнить определённую операцию. Он хранит внутренний счётчик доступных «разрешений» (permits) и предоставляет два основных метода:
* Acquire (захватить) — уменьшает счётчик на 1. Если счётчик равен 0, горутина блокируется до тех пор, пока не появится свободное разрешение.
* Release (освободить) — увеличивает счётчик на 1 и будит одну из ожидающих горутин.

- Семафоры особенно полезны, когда нужно ограничить число одновременных соединений к базе данных, количество параллельных HTTP-запросов к внешнему API или число горутин, работающих с тяжёлым ресурсом.
