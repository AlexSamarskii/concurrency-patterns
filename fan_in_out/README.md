### **Fan-In** 

- паттерн, который объединяет (мультиплексирует) несколько входных каналов с данными от разных источников в один выходной канал. Он является логическим продолжением Fan-Out, когда мы сначала распределяем работу по множеству воркеров, а затем собираем результаты от всех в единый поток
- Каждый воркер пишет результаты в свой собственный выходной канал. Функция fanIn запускает для каждого такого канала горутину, которая пересылает все приходящие значения в общий канал out. После завершения всех горутин пересылки (отслеживается через sync.WaitGroup) общий канал закрывается
____
### **Fan-Out**

- Fan-Out реализован тем, что мы запускаем несколько горутин-воркеров, и все они читают из одного и того же входного канала inputCh. Канал inputCh автоматически распределяет задачи между воркерами: когда один воркер читает значение, оно удаляется из канала, и другие воркеры его уже не получат. Это безопасно и идиоматично
____

### Синхронизация и закрытие каналов

1. Генератор закрывает свой канал после отправки всех данных
2. Воркер закрывает свой выходной канал, когда входной канал опустеет и закроется (цикл for range завершится)
3. fanIn закрывает общий канал после того, как все каналы-источники будут вычитаны и закрыты
