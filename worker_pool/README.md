**Worker Pool** — это паттерн конкурентного программирования, при котором создаётся фиксированное количество горутин-воркеров, обрабатывающих задачи из общей очереди (канала).

Позволяет:
- Ограничить количество одновременно работающих горутин.
- Эффективно распределять нагрузку.
- Избежать чрезмерного создания горутин при большом потоке задач.
-----
1. Каналы задач и результатов
taskCh := make(chan Task, 10) — буферизированный канал. Отправитель не блокируется, пока не заполнит весь буфер, что сглаживает пиковые нагрузки.
resultCh — позволяет воркерам возвращать результаты. Главная горутина читает их после завершения отправки задач.
2. Закрытие каналов
close(taskCh) — обязателен. Без него цикл for task := range taskCh в воркерах никогда не завершится, и программа зависнет.
close(resultCh) — выполняется после wg.Wait(), чтобы главная горутина знала, когда можно прекратить чтение результатов.
3. WaitGroup
Каждый воркер вызывает wg.Done() при выходе.
Главная горутина может либо wg.Wait() и потом читать результаты (если результат неважен в реальном времени), либо запустить отдельную горутину для ожидания и закрытия resultCh, как в примере.
