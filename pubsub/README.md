**Publish-Subscribe (издатель-подписчик)** — это поведенческий паттерн, обеспечивающий слабую связанность компонентов. Издатели публикуют сообщения в определённые топики (темы), а подписчики, выразившие интерес к этим топикам, получают сообщения. Брокер управляет подписками и маршрутизирует сообщения.

* Message — сообщение с темой и произвольными данными
* Subscriber — подписчик с уникальным ID и каналом, в который будут приходить сообщения
* Broker — хранит отображение topic → []*Subscriber и мьютекс для защиты конкурентного доступа
____

#### Методы

* Subscribe(topic string, subscriber *Subscriber) — добавляет подписчика в список для указанной темы
* Unsubscribe(topic string, subscriber *Subscriber) — удаляет подписчика из темы
* Publish(msg Message) — рассылает данные сообщения всем подписчикам темы, записывая в их каналы
* Update() у подписчика — бесконечный цикл чтения из канала и обработки сообщений
